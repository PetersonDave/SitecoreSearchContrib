Help

Here are quick steps to follow if you are having trouble with getting searching to work properly.

1. Verify that your index is getting built

If your index is not getting updated, it's probably an issue with your Sitecore Search configuration, not the module.

You may want to run through these troubleshooting steps to confirm.

2. Make sure that things are getting indexed

2.1 Check the index files under /data/indexes folder.

2.2 Check the Sitecore logs for any errors during indexing.

2.3 Use Luke. It's awesome.

The tool will let you open up your index and browse it. Here is the link to the download page

3. Make sure your quieries are properly built.

You may be getting no hits because of malformed queries. To check for that, follow these steps:

3.1 Activate DEBUG mode

For that, locate the Sitecore LogFileAppender in web.config and change it's priority to DEBUG:

    <root>
      <priority value="DEBUG"/>
      <appender-ref ref="LogFileAppender"/>
    </root>

This will enable query tracing in Sitecore log file, which you can then paste into Luke to test the query.

Overall, when querying, you have to adhere to Lucene's rules. For example, you can have only one search parameter with MustNot condition, etc.
